## §0 結論與優先序

七個 LLM 表面中,**master 評分提示詞是全系統品質的槓桿點且缺陷最重**(無評分準則
→ Loop 3 校準在噪音上學習)。其次是健檢卡的**預測指令模糊**(metric/幅度/flat 時機
未指示 → 戰績可比性差)。資料面最大機會是 180 天日線降採樣(健檢輸入 40%+ 的 token)。
技術訊號與 F&G 屬「缺料」而非「缺提示詞」——要先補計算核心變數(不變式 #1:訊號一律
本地計算,LLM 只判讀)。

## §1 現行組裝事實(審查基準)

- 組裝硬順序(spec 4.0):系統提示詞(可關)→ 策略 1..n → 生效校準版 → (on_alert 短欄)
  → F1 JSON schema 契約(seam 層,2026-07-05 修復後一律附加)。
- 真實負載(測試站實測):週報 est 2,469 tokens(系統 259 字元+策略渲染 9,615 字元);
  健檢每檔 est 2,067(策略渲染 8,007 字元)。實際計費 input ~4.3k/call(JSON 契約
  ~400 tokens+中文 tokenizer 差異)。健檢 9 檔/週 ≈ 40k input tokens。
- 變數 24 個可組;快取指紋=日錨定渲染(同日同資料不重扣費)。

## §2 表面逐一審查

### 2.1 全域系統提示詞(方向性判讀版,6 原則)
**目標**:全域行為約束(語言/幣別/紅漲綠跌/引用數字/建議邊界/新鮮度誠實)。
**問題**:
- 🟡 無輸出結構指導:title/summary/body_md 的分工全靠模型自由發揮(title 應可掃讀、
  summary 應獨立成立、body 才展開)——首頁與行動端的閱讀體驗取決於此。
- 🟡 無「信心校準」心態指導:confidence 是 Loop 2/3 的量化錨,應明示「信心值=你對
  預測命中的真實概率估計,寧低勿高;過度自信會被校準系統記錄」。
- 🟢 tags 無詞彙表(模型隨意發明標籤,無法聚合篩選)。
- 🟢 「每則洞察精簡扼要」與健檢五節結構有張力——長度基準應由各策略層自定。

### 2.2 持倉週報策略(v2026-07-05 初版)
**目標**:低頻高價值的全組合敘事回顧,易讀、被信任。
**問題**:
- 🟡 五節全是「描述指令」,缺**合成指令**:未要求跨節連結與「本週最重要的一件事」
  ——目前產出像五段獨立播報,缺總編輯視角。
- 🟡 未定調讀者情境(長期投資人週檢視,非短線),語氣易飄。
- 🟢 除息日曆與年度股利預估重疊(兩變數可合併敘述指令)。
- 節食:holdings_json 與 allocation_json 的權重欄重複;週報其實不需要 180 天級明細。

### 2.3 個股健檢策略(v2026-07-05 初版)
**目標**:單標的多面向健檢+可驗證方向預測(戰績/校準的資料來源)。
**問題**:
- 🔴 **預測指令模糊**:只說「未來兩週的方向預測與信心值」。Prediction schema 有
  metric(price_change/volatility/relative)、target_pct(可選幅度)、horizon_days
  ——未指示 metric 怎麼選(應預設 price_change)、target_pct 何時給、direction=flat
  的使用時機(±0.5% 帶內)。模型自由發揮 → 每張卡的預測形狀不一 → Loop 2 可比性差。
- 🟡 你的新要求「建倉/減倉邏輯依據」未入指令:現只要偏多/偏空/觀望+條件情境,
  未要求扣回「長期持倉的加減碼參考框架」(等 §4 技術訊號變數到位後一併寫入)。
- 🟡 非台股(US/MY)籌碼節跳過後,未告訴模型該用什麼替代支柱(技術+價格成本+環境)。
- 節食:price_history_json 180 點 ≈ 3.4k 字元,佔輸入 40%+;且 MA/波動/回撤已是
  摘要——原始序列的邊際價值低。

### 2.4 master 評分(_SCORE_SYSTEM)——🔴🔴 全系統最高優先
**目標**:對到期卡打敘事分 0–100+miss 判定 → **Loop 3 校準的學習訊號**。
**問題**:
- 🔴 **無評分準則**:無維度(數字引用正確性/方向 vs 實際/條件情境是否觸發/過度自信)、
  無分數錨(90=?50=?20=?)、無 miss 定義(方向相反?捏造數字?)——同一張卡不同次
  評分可能差 30 分,校準迴路在噪音上學習。
- 🟡 note 無要求引用具體證據(Loop 3 靠 note 改寫規則,空泛 note=空泛校準)。
- 🟡 eval_prompt(每任務自訂檢驗模板)無官方預設——健檢類應有官方評分補充模板。

### 2.5 master 校準(_CALIBRATION_SYSTEM)
**目標**:依失誤樣本+信心分桶改寫校準規則(安全鎖防越權)。
**問題**:
- 🟡 安全鎖寫「全文總字數不得超過上限」但**沒給數字**(建議明定,如 ≤600 字)。
- 🟡 失誤樣本只餵 `insight id + note`——沒有卡片摘要、預測內容、實際結果,校準大師
  是在「二手筆記」上改規則(餵料在 scoring 側組裝,屬資料面修改)。

### 2.6 master 驗證(_VALIDATE_SYSTEM):🟢 合格(denylist 先行+單次審查,結構清楚)。

### 2.7 AI 對帳單文字解析(agents._PROMPT)
**目標**:自然語言→合法交易草稿(帳戶目錄已於 2026-07-05 注入)。
**問題**:
- 🟡 日期規則未定:「7/3」的年份推定(今年?最近過去?)、未來日期應拒;
- 🟡 只有單筆範例,多筆混合文字(常見貼上整段對帳單)無 few-shot;
- 🟢 手續費資訊只能進 note(AiDraft 無 fee 欄,費用由費率引擎推算)——實際成交費
  與規則不符時的落差,記錄待議(超出提示詞範圍)。

## §3 資料節食分析(與提示詞一體優化)

| 變數 | 現況 | 建議 |
| --- | --- | --- |
| price_history_json | 180 點日線全量(~3.4k 字元/檔) | **近 30 日日線+其餘週線收盤**(~70 點,約 -55% token);技術摘要已有 MA/波動變數承擔 |
| holdings_json | 全欄位(週報用) | 週報版精選:symbol/股數/調整均價/現價/未實現/權重/回本率(去除重複與內部欄位) |
| symbol_detail_json | 全檔(含全部配息史+交易事件) | 近 12 個月事件+彙總統計(長史壓縮) |
| dividends_json(週報未用)/ex_div+projection | 兩變數重疊 | 提示詞層合併敘述即可,變數不動 |
| freshness/fx_rates | 輕量 | 不動 |

原則:壓縮**表示形式**不是刪資訊——降採樣週線仍覆蓋 180 天趨勢;所有數字仍由計算核心產出。

## §4 技術訊號缺口(新要求:市場標準技術判讀,輔佐建倉/減倉)

**現有**(portfolio/technicals.py,純 Decimal):MA20/60/120+乖離、30 日年化波動、
90 日最大回撤、價格 vs 成本。**缺**(市場標準、適合長期持倉加減碼判讀):
1. 均線交叉事件:MA20↕MA60 黃金/死亡交叉+距今天數(趨勢轉折錨點);
2. RSI(14)(超買超賣區間);
3. 52 週高/低距離(位階感);
4. 趨勢結構:近 60 日高低點抬升/下移(HH-HL / LH-LL);
5. 量能:`prices` 表**已有 volume 欄**——需先確認各 provider 是否回填(探測項),
   有料才做量價背離/放量突破類訊號。
**落地方式**:全部為 portfolio/technicals.py 純函數(不變式 #1)→ 新變數
`technical_signals_json`(整合輸出,避免變數爆炸)→ 健檢策略提示詞加入「以技術訊號
輔佐建倉/減倉條件判讀」指令。**需 mini-spec + 簽核後實作**(計算核心變更)。

## §5 CNN Fear & Greed 信號分析審查(新要求)

**現況**:VIX 有本地分區(<15 low/15-25 normal/25-35 elevated/≥35 high);
**F&G 只是 passthrough**(CNN 的 score+rating 原樣進提示詞),無本地信號分析。
**缺口與建議**:
1. 本地分區(CNN 標準五區:0-24 extreme fear/25-44 fear/45-55 neutral/56-75 greed/
   76-100 extreme greed)——不依賴 CNN rating 字串格式;
2. **趨勢**:external_snapshots 是 append-only,歷史快照已在庫——可推導 7 日變化
   (由 fear 轉 greed 的過程比絕對值更有信號);
3. 使用指引入提示詞:極端區間的反向指標語境(extreme fear 歷史上偏機會、extreme
   greed 偏風險——僅作環境定調,不得直接轉譯為買賣訊號,守方向性判讀邊界)。

## §6 官方模板庫(載體設計概要,Phase 2 工程)

- 官方模板 = **程式內建常數(英文檔案、中文內容)+ 版本號**:系統提示詞/週報策略/
  健檢策略/評分 rubric/校準系統詞/解析詞全套;
- 出廠 seed 即官方版;設定頁每個提示詞旁「重置回官方版 vX」+「從模板庫新增」;
- 使用者編輯=編輯自己的副本,官方版永遠可回;
- `prompt_version` 由常數 "v1" 改為真版本標記(評測分組/戰績歸因用);
- 離線 A/B:固定合成快照下渲染新舊版+試產卡,依檢核表(數字引用正確/無捏造/結構/
  可行動性/token 成本)評審,勝者晉官方版。
