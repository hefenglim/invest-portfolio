分析師共識變數：目標價＋評級分布進入變數系統

1為什麼做

健檢卡目前只有技術面與籌碼面，沒有任何「市場對這檔股票的集體看法」。分析師目標價與評級分布是估值疊加的基準線（藍圖 P3／Phase 6 的前置資料）。

數字一律 finance API 取得、本地存檔，LLM 只詮釋——invariant #1 合規（藍圖 D9 慣例）。

2探測結果（2026-07-08 實測，超出藍圖預期）

yfinance analyst_price_targets＋recommendations_summary 三市場全通：AAPL 42 位分析師、2330.TW 33 位、連 1155.KL 也有 19 位（藍圖原估 MY 稀薄）。無覆蓋的個股（如部分 TPEx 小型股）仍需誠實降級。

3欄位提案（最小可靠集）＋管線

consensus_json（per_symbol，snapshot 模式）：
{
 "as_of": "2026-07-09",
 "price_targets": { "current": "2465.00",
   "mean": "2819.8484", "median": "2780.00",
   "high": "3800.00", "low": "2051.00" },   ← Decimal 字串（報價幣別）
 "ratings": { "strong_buy": 9, "buy": 23,
   "hold": 1, "sell": 0, "strong_sell": 0,
   "total": 33 },                            ← 本月評級分布
 "ratings_prev_month": { … },                ← 上月（看趨勢）
 "rating_score": "1.76",                     ← 1=強力買進…5=強力賣出（本地加權計算）
 "upside_vs_mean_pct": "0.1440",             ← (mean−current)/current（本地計算）
 "source": "yfinance"
}
① 來源模組：pricing/consensus_source.py，走既有 snapshot 慣例（比照籌碼/情緒五件套：SOURCE_INFO＋冪等 upsert＋健康降級）。不用重型的 Ticker.info，只打兩個輕端點。價格類數字沿用 4dp 噪音帽。
② 排程＋手動：每日一次 consensus_daily job（比照 sentiment/index 的既有時段慣例）；沿用既有手動觸發機制供 demo 驗證（demo 排程停用）。
③ 變數註冊：consensus_json 上 registry（32→33 變數），經 external_vars 餵入（籌碼前例，架構層向不變）；vars.js／var-count 契約測試同步。
④ 健檢 v2.4：官方個股健檢模板 v2.3→v2.4，引用共識（目標價區間、評級分布、與現價差距），無覆蓋時明講「無分析師覆蓋」；官方模板位元組凍結＋版本戳慣例不變。

  • 無覆蓋個股誠實降級（unavailable＋原因），永不編造
