1為什麼做

prices.volume 欄位存在但全庫 0 筆有資料——黃金交叉的量能確認（實證勝率：高量 72% vs 低量 54%）完全不可用；歷史只有 365 天，MA200／52 週視窗／未來回測的樣本都不誠實。

3做四件事

① 量能接線（純接線）：yfinance provider 的 history／latest 補讀 Volume 欄 → PriceRow.volume。儲存欄位與 upsert 早已預留，整數字串入庫（量不是錢，不受 2dp 規則）。
② 台股歷史備援：FinMind provider 擴充 QUOTE_HISTORY（收盤＋量能），排在 yfinance 之後的 fallback 鏈——順帶解掉「歷史只有 yfinance 單點」的風險項。維持現有 token 門控。
③ 歷史深度 5 年（你的定案，取代藍圖 3 年建議）：單一 config 設定 history_backfill_days=1825（env 可覆寫），統一取代散落的兩處 365；智慧視窗「延伸到首次買入日」的邏輯不變。儲存量約 1,250 列/標的，微不足道。
④ 讀取路徑＋深回填：get_price_history 回傳補上 volume；洞察生成餵入 ctx.volumes（與收盤對齊，探測門控不變）；沿用既有回填 action 一次深回填——冪等 upsert 會把舊列的空 volume 一併補上。
