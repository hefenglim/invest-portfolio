## 一、 馬來西亞股票 (Bursa Malaysia) 普通訂單

*   **基礎計算公式**：`總費用 = 佣金 + 平台費 + 清算費 + SST + 印花稅`

### 1. 基礎共通收費
| 收費項目 | 收費費率 / 計算公式 | 條件限制 (Min / Max) | 收取情境 (Flag) |

**說明**：此處費用與稅務是由**美國聯邦政府、監管機構 (SEC/FINRA)** 或是**美國國稅局 (IRS)** 徵收。無論使用 Moomoo 還是 Charles Schwab，交易時均會強制觸發相關扣款。

| 規費/稅務項目 | 收取單位 | 計算公式 / 費率基準 | 收取情境 (Flag) |

# 肆、 軟體開發計算引擎實作指引 (Developer Integration Notes)

### 1. 模組化與可設定性 (Configuration Over Hardcoding)
美國 SEC 規費和 FINRA 的 TAF 費率**每年都會動態調整**。請務必在資料庫或系統 Config 檔中，將這些費率設為可配置變數，切勿寫死在函式中：
*   `US_SEC_FEE_RATE = 0.0000206`
*   `US_TAF_FEE_RATE = 0.000195`

### 2. 精確度處理 (Precision Control)
為防止金融計算中的浮點數精度誤差，系統在累加與折算手續費時，**禁止使用原生 `Float` 或 `Double`**。
*   **JavaScript/TypeScript**：推薦使用 `decimal.js` 或 `bignumber.js`。
*   **Python**：推薦使用 `decimal` 模組。
*   **Java**：必須使用 `BigDecimal`。

### 3. 多幣種印花稅計算 (Cross-Currency Stamp Duty)
在計算美股交易的「馬來西亞印花稅」時，須嚴格執行以下流程（偽代碼範例）：

```javascript
function calculateUSStampDuty(tradeAmountUSD, usdToMyrRate, assetType) {
   // 1. 換算為馬幣
   let tradeAmountMYR = tradeAmountUSD * usdToMyrRate;
   // 2. 每 RM 1,000 向上取整，乘以 RM 1.00
   let calculatedStamp = Math.ceil(tradeAmountMYR / 1000) * 1.00;
   // 3. 判斷上限 (ETF / REITs 上限為 RM200; 一般正股上限為 RM1,000)
   let maxCap = (assetType === 'ETF' || assetType === 'REIT') ? 200.00 : 1000.00;
   return Math.min(calculatedStamp, maxCap);
}
```

### 4. 下單方向判斷與防呆 (Side & Channel Flags)
在寫入手續費扣減明細 (Transaction Fee Ledger) 時，必須綁定條件觸發：
*   `if (order.side === 'SELL') { applySecFee(); applyTafFee(); }`
*   `if (order.channel === 'BROKER_ASSISTED') { applySchwabAssistedSurcharge(); }`

# 台灣股市交易成本計算指南（群益證券：2.3折/先收後退專案）

本指南適用於「交割當下無折扣（收取 0.1425% 原價），日後（通常為次月）再退回折讓款」之計費模式。

## 一、 交易成本與退款機制

| 項目 | 交割當下（T+2 日） | 日後退款（次月月中） |
| :--- | :--- | :--- |
| **買進手續費** | 先收取 **原價（0.1425%）** | 退回 **77% 差額**（即實收 2.3折） |
| **賣出手續費** | 先收取 **原價（0.1425%）** | 退回 **77% 差額**（即實收 2.3折） |
| **證券交易稅** | 直接扣除 **0.3%**（或 ETF 0.1%） | 屬於政府稅收，**無任何退款** |

> ⚠️ **重要提醒**：因交割當下採「無折扣」扣款，請務必確保交割銀行帳戶內有足夠的**原價**手續費資金，避免交割金額不足。

## 二、 核心計算公式

### 1. 交割當下（T+2 日）扣款金額
*   **買進交割金額** = 買進成交價價金 + 買進原價手續費
    > 買進原價手續費 = 買進成交價金 × 0.1425% （未滿 20 元以 20 元計）
*   **賣出交割金額** = 賣出成交價價金 - 賣出原價手續費 - 證券交易稅
    > 賣出原價手續費 = 賣出成交價金 × 0.1425% （未滿 20 元以 20 元計）
    > 證券交易稅 = 賣出成交價金 × 0.3%

### 2. 次月退佣（折讓金）回饋計算
*   **單筆退款金額** = 原價手續費 × (1 - 0.23) = **原價手續費 × 77%**
    *   *註：各券商計算退佣可能採單筆計算後加總，或全月總成交量加總後再乘以折數，遇小數點皆採無條件捨去。*

## 三、 實際試算範例（以 2.3 折、日後退款為例）

**情境**：以每股 100 元買進 1 張（1,000股）一般股票，並在數日後以 110 元賣出。

### 【階段一：交易當下交割金額（T+2）】

*   **買進日交割（付出）：**
    - 成交價金：100,000 元
    - 原價手續費：100,000 × 0.1425% = 142.5 ➡️ **142 元**
    - **實際帳戶扣款：100,142 元**

*   **賣出日交割（回收）：**
    - 成交價金：110,000 元
    - 原價手續費：110,000 × 0.1425% = 156.75 ➡️ **156 元**
    - 證券交易稅：110,000 × 0.3% = **330 元**
    - **實際帳戶入帳：110,000 - 156 - 330 = 109,514 元**

*   **交割當下帳面利潤**：109,514 - 100,142 = **9,372 元**

### 【階段二：次月折讓款退回（月退）】

次月月中，群益證券會將 7.7 折的差額退回至您的交割帳戶：

*   **買進手續費退款**：142 元 × 77% = 109.34 ➡️ **109 元**
*   **賣出手續費退款**：156 元 × 77% = 120.12 ➡️ **120 元**
*   **次月退款總額：109 + 120 = 229 元**

### 📊 最終損益總結（完全體）

*   **實際總手續費（2.3折）**：(142 - 109) + (156 - 120) = 33 + 36 = **69 元**
*   **實際總交易成本**：實際手續費 (69) + 證交稅 (330) = **399 元**
*   **最終實賺利潤**：帳面價差 (10,000) - 實際成本 (399) = **9,601 元**
    *(相較於無折扣，2.3 折專案在此筆交易幫您省下了 229 元的成本！)*
