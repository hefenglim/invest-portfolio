ntfy
主題名即密碼，請勿外流；手機 ntfy app 訂閱此主題即可收到推播。
僅受保護主題需要；以遮罩顯示。
📖 設定步驟（約 1 分鐘）
  1. 手機安裝 ntfy app（App Store／Google Play，免費、無需註冊）。
  2. 在本卡按「啟用」，然後按「複製」取得主題名（主題名＝密碼，勿外流）。
  3. 手機 ntfy app → ＋ Subscribe to topic → 貼上主題名 → 訂閱。
  4. 回本頁按「傳送測試訊息」→ 手機應立即收到。

沒收到？「測試成功」代表已送達伺服器——請確認手機訂閱的主題名完全相同、且已允許 ntfy 的通知權限。伺服器預設 ntfy.sh（免費）；自架 ntfy 者改填自己的網址。

Telegram
向 @BotFather 建立 bot 取得；以遮罩顯示。
需先在 Telegram 對你的 bot 傳送 /start（bot 不能主動發起對話）；chat_id 為數字，私訊 @userinfobot 可查，群組 id 為負數。「chat not found」= 尚未 /start 或 chat_id 錯誤。
📖 設定步驟（約 3 分鐘）
  1. Telegram 搜尋 @BotFather → 傳送 /newbot → 依指示命名 → 取得 Bot Token 貼入上方欄位。
  2. 私訊 @userinfobot，它會回覆你的數字 Chat ID，貼入上方欄位。
  3. 對你的新 bot 傳送一次 /start（必要！bot 不能主動發起對話，漏了這步會出現「chat not found」）。
  4. 按「啟用」→「儲存」→「傳送測試訊息」→ Telegram 應收到訊息。

要發到群組：把 bot 加入群組，Chat ID 用群組的（負數）。

Email
以遮罩顯示；建議使用應用程式密碼。
📖 設定步驟（以 Gmail 為例）
  1. SMTP 主機 smtp.gmail.com、連接埠 587、加密 STARTTLS（若用 465 則選 SSL——埠與加密必須成對，配錯會逾時）。
  2. Gmail 必須用「應用程式密碼」：Google 帳戶 → 安全性 → 開啟兩步驟驗證 → 應用程式密碼 → 產生 16 碼貼入「密碼」。一般登入密碼會被拒（錯誤 535）。
  3. 帳號＝完整信箱；寄件者通常需與帳號相同；收件者填要接收警示的信箱。
  4. 按「啟用」→「儲存」→「傳送測試訊息」→ 收件匣（含垃圾郵件夾）確認。

其他信箱服務：向提供者查 SMTP 主機／埠；多數同樣要求應用程式專用密碼。
